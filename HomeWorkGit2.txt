   
Описание задания.

 1 уровень сложности: 1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
2.Как вернуть файл из индекса в рабочий каталог?
3.Как отменить ошибочный commit ?
4.Как создать новую ветку и переключится в нее?
5.В чем отличение между fast-forward merge  and non fast-forfard merge?
6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
7.Можно ли удалить ветку? Как? 

В ответах на вопросы 1-4, 7 необходимо указать конкретные команда git. 
На 5 и 6 вопросы дать объяснение своими словами.


1. git checkout file.txt file2.txt
2. git restore --staged file.txt
3. git reset HEAD изменить кормит на "зеркальный"
git reset HEAD~[число - количество коммитов на которые надо откатиться назад]
4. git checkout -b [name branch] создает и сразу переключает на ветку
5. fast-forward merge - при линейной истории коммитов, слияния происходит с помощью перемещения указателя ветки, используют когда на вы сливаете с основно веткой не имеющей коммитов
non fast-forfard merge - когда основная ветка имеет коммиты, тогда git создает новый кормит слияния, который объединяет изменения с веток
6. Когда хотят слить вместе изменения в одном файле(в одной части файла кода). Git сщщбщит об этом при проще сливания веток. Надо вручную внести правки в конфликтный файл через текстовой редактор (git пометить места конфликта маркером) - внести правки, удалить изменения, выбрать нужные изменения и тд.,удалить маркер, выйти с сохранением, затем git add файл и git commit
7.git branch -d [name branch] - не забыть при этом перейти на другую ветку
